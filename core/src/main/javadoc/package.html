Hibernate JavaDoc documentation.
The following are considered to define the "core API" of Hibernate, meaning it is fully intended that they be exposed to application code:
  • {@link org.hibernate.Hibernate}
  • {@link org.hibernate.Session}
  • {@link org.hibernate.SessionFactory}
  • {@link org.hibernate.Transaction}
  • {@link org.hibernate.Query}
  • {@link org.hibernate.Criteria}
  • {@link org.hibernate.ScrollableResults}
  • {@link org.hibernate.cfg.Configuration}
  • {@link org.hibernate.criterion.Restrictions}
  • {@link org.hibernate.criterion.Order}
  • {@link org.hibernate.criterion.Example}

The Extension SPIs are intended to be used by application programmers to extend Hibernate functionality. None of these interfaces are intended to be called by the application - they are called internally by Hibernate. These contracts are less stable than the Core API. The safest way to extend functionality is to contribute extensions back to the project.
